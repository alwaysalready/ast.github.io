Sending HTML formatted emails using BuddyPress and BBPress

Posted on 12/06/2020 (21/01/2021) by Alex Taylor

I’ve spent quite a long time try­ing to fig­ure out how to send html for­mat­ted emails on a com­mu­ni­ty Word­Press site, using Bud­dy­press and bbpress. It’s tak­en a bit of dig­ging and a com­bi­na­tion of sources so I thought I’d share my final solution.

The problem

I run a Word­Press web­site for my local neigh­bour­hood (of approx 200 peo­ple). The com­mu­ni­ty makes good use of the bbpress enabled dis­cus­sion forum. All mem­bers are auto­mat­i­cal­ly sub­scribed to the forum and receive email noti­fi­ca­tions when mes­sages are posted.

At first, I relied on Bud­dy­press’s built-in use of PHP­Mail­er to send the email noti­fi­ca­tions. I found, how­ev­er, that my ser­vice provider was lim­it­ing the num­ber of emails that could be sent per day and that many of the mes­sages were end­ing up in peo­ple’s junk mail fold­ers. As a result, a lot of peo­ple weren’t get­ting the notifications.

My solu­tion to this was to sub­scribe to a sep­a­rate email ser­vice and use the WP Mail SMTP plu­g­in to send mes­sages via the alter­na­tive, more reli­able mail server.

Plaintext emails

As is so often the case in Word­Press, this trig­gered anoth­er prob­lem. To send mes­sages through exter­nal mail servers, Word­Press needs to use the wp_mail() func­tion. And here’s the rea­son for this blog post: Bud­dy­Press and it’s younger sib­ling bbpress default to plain­text emails when wp_mail() is used. In Snip­pet A from one of Bud­dy­press’s core files you’ll see the cause of this prob­lem. At 3201, you’ll see ‘content_plaintext’. This sets the mail for­mat to be plain­text when wpmail() is used.

There are a few rec­om­men­da­tions for chang­ing this code in the core files. How­ev­er, as many who use Word­Press will know this is not advis­able, not least because you’ll lose the changes on any updates to the plu­g­in. As an exam­ple see Snip­pet B where the sug­ges­tion is to replace the offend­ing line above with: $email->get_template( 'add-content' ).

The solution

The solu­tion requires three steps. Before we begin, we need to either open or cre­ate a file called bp-custom.php in the wp-content/plugins folder.

1. Use wp_mail

In this file, we need to tell Bud­dy­press to use wp_mail() using the bp_email_use_wp_mail filter:

// Set BP to use wp_mail and so send via Google mail
add_filter( 'bp_email_use_wp_mail', '__return_true' );

2. Set to HTML

Next, we have to tell WordPress/Buddypress to set the con­tents of the mes­sage to be in HTML rather than plain­text. The remove_filter is for redun­dan­cy and then you are call­ing a func­tion, set_html_content_type, that tells wp_mail to use HTML text.

// Set messages to HTML
remove_filter( 'wp_mail_content_type', 'set_html_content_type' );
add_filter( 'wp_mail_content_type', 'set_html_content_type' );
function set_html_content_type() {
    return 'text/html';
}

3. Use the HTML template

Final­ly, and this is the real­ly impor­tant part, we need to tell Bud­dy­press to use the HTML tem­plate rather than the plain­text con­tent. This over­writes the code in Snip­pet A, line 3201 above.

// Use HTML template
add_filter( 'bp_email_get_content_plaintext', 'get_bp_email_content_plaintext', 10, 4 );
function get_bp_email_content_plaintext( $content = '', $property = 'content_plaintext', $transform = 'replace-tokens', $bp_email ) {
	if ( ! did_action( 'bp_send_email' ) ) {
		return $content;
	}
	return $bp_email->get_template( 'add-content' );
}

4. Bonus

Final­ly, just a small bonus. You may want to send Bud­dy­Press emails with a seper­ate email address. In my case, I want­ed to make it clear that mes­sages from the forum could­n’t be replied to using email, so I’ve used a ‘no-reply’ address and name.

add_filter('wp_mail_from','noreply_from');
function noreply_from($from) {
  return '[email protected]_DOMAIN.org'; //Replace 'YOUR_DOMAIN.org' with email address
}

add_filter('wp_mail_from_name','noreply_from_name');
function noreply_from_name($name) {
	return 'YOUR_DOMAIN No-Reply'; //Replace 'YOUR_DOMAIN No-Reply' with the from name
}

So, in sum­ma­ry, the bp-custom.php should look like Snip­pet C (if you have no oth­er code in it).

Hope this works and is use­ful for others.

Notes:
1. There are a few rea­sons for this improved reli­a­bil­i­ty that I won’t go into here — WP Mail SMTP offer a rea­son­able expla­na­tion.
2. See: \wp-content\plugins\buddypress\bp-core\bp-core-functions.php from about line 3196.
3. I’m assum­ing you have access to your Word­Press site’s root direc­to­ry, via ftp for example.
